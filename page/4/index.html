发表于 2024-01-01 Valine：

问题描述

最近做需求，有个功能需要导出。技术用的是 hutool 工具的导出。发现大家导出都是后端根据前端页面写死了。这种写法太不灵活了，太蠢了。于是想着写个封装导出功能的工具类。由前端给出需要导出的字段。后端动态实现导出功能。

问题分析

以下是导出代码 后端写死了

优化思路

  1. 先分析hutool 导出的api。重要的api多表头合并 ExcelWriter.merge(int firstRow, int lastRow, int firstColumn, int lastColumn, Object content, org.apache.poi.ss.usermodel.CellStyle cellStyle).
    前端用的是react。 表头数据是一个树。 所以，我们需要知道一个树的深度。 这个深度就是表头的宽度。也就是合并的行数。 我们也需要知道树叶子节点的宽度。这个是确定多表头的列数。 由以上思想 先建一个column 类
  1. 再建一个操作树的工具类
  1. 导出工具类
  1. 前端请求类
  1. hutool 导出APi 文档地址

优化结果

后端不需要在写死导出表格的数据, 几行代码就解决导出。

进一步优化空间

  1. idmc 转换，运用反射技术实现。 后端不再需要去库里查询字典。
  2. 算法优化。 树的广度遍历与深度遍历都是用的递归。 还有进一步优化空间。
  3. 性能优化。 可以根据导出数据的大小。使用多线程去导出。或换成 BigExcelWriter 去导出。
    以上是我觉得还可以优化的地方。因为业务繁忙，以后有时间再去优化。

使用 umi 组织管理项目

umi 是什么

Umi，中文可发音为乌米，是可扩展的企业级前端应用框架。Umi 以路由为基础的，同时支持配置式路由和约定式路由，保证路由的功能完备，并以此进行功能扩展。然后配以生命周期完善的插件体系，覆盖从源码到构建产物的每个生命周期，支持各种功能扩展和业务需求。
官方文档： https://umijs.org

Umi.js 是一个基于 React 的企业级前端框架，提供了一套约定优于配置的机制来组织和管理项目。以下是 Umi.js 如何组织和管理项目的关键点：

  1. 目录结构
    src/pages：这是 Umi.js 的核心目录，Umi.js 会根据这个目录下的文件和文件夹自动生成路由。每个文件对应一个路由页面，每个文件夹可以包含多个页面组件。

src/layouts：用于存放布局组件。你可以在这里定义应用的整体布局结构，并在路由配置中使用这些布局。

src/models：用于存放应用的全局状态管理文件，通常与 Umi.js 的插件 Dva 一起使用。

src/services：用于存放与后端 API 交互的代码，通常是一些请求函数。

src/assets：用于存放静态资源，如图片、字体等。

src/components：用于存放应用的通用组件。

src/app.js 或 src/app.ts：用于配置全局的应用设置，比如路由守卫、全局状态等。

  1. 路由管理
    Umi.js 使用约定式路由，通过 src/pages 目录自动生成路由。每个页面组件会根据其文件路径自动生成对应的路由路径。你也可以使用配置式路由，通过配置文件手动定义路由。

  2. 插件系统
    Umi.js 提供了强大的插件系统，允许开发者通过插件扩展框架功能。常见的插件包括：

Dva：用于状态管理。
Ant Design：用于 UI 组件库集成。
国际化：支持多语言配置。
权限管理：用于控制页面访问权限。 4. 配置文件
Umi.js 提供了灵活的配置文件，通常是 .umirc.js 或 config/config.js，用于配置路由、插件、主题、构建选项等。

  1. 构建与部署
    Umi.js 使用 Webpack 进行构建，提供了多种优化和定制选项。你可以通过 umi build 命令生成生产环境的构建文件。

  2. 开发体验
    热重载：Umi.js 支持热重载，开发时修改代码会自动更新页面。
    调试工具：提供了 Umi UI 工具，可以可视化管理项目配置和插件。
    Umi.js 的设计目标是提供一个高效、易用的开发环境，通过约定优于配置的理念减少开发者的配置负担，同时提供灵活的扩展点以满足不同项目的需求。

config 配置

以上主要是配置了以下信息

  1. 插件配置
    使用了 umi-plugin-react 插件，配置了 Ant Design、Dva、目标浏览器版本等。
    根据环境变量 APP_TYPE，可能会添加 umi-plugin-ga 插件用于 Google Analytics。
  2. 环境变量
    定义了一些环境变量，如 APP_TYPE、IS_DEV、SERVER_CONTEXT 等，用于在项目中使用。
    根据环境变量 process.env.NODE_ENV 判断当前环境是开发环境还是生产环境。
  3. 代理配置
    使用 proxy 属性配置了多个代理，用于将前端请求转发到不同的后端服务。
    每个代理配置项包含 target、changeOrigin、logLevel 等属性。
  4. 主题配置
    使用了 Ant Design 的主题定制功能，通过 theme 属性配置主题颜色和样式。
    根据 process.env.THEME_COLOR 的值选择不同的主题配置。
  5. 路由和静态资源配置
    使用 routes 属性指定路由配置。
    使用 publicPath 和 base 属性配置静态资源和路由的前缀。
  6. 其他配置
    ignoreMomentLocale：设置为 true，忽略 moment.js 的本地化文件。
    lessLoaderOptions：配置 Less 的加载选项。
    cssLoaderOptions：配置 CSS 模块的选项，指定如何生成 CSS 类名。
    manifest：配置生成的 manifest 文件的基本路径。
  7. 自定义插件
    在开发环境下，可能会根据 enableDevLoginModal 的值添加自定义插件 ./config/devLogMiddlewarePlugin.js。

dva 是什么

Dva.js 是一个基于 React 和 Redux 的轻量级框架，适用于构建数据驱动的前端应用。 它的核心特点包括：简化 Redux 的使用、内置异步处理和路由管理、强大的插件机制。Dva.js 通过将 Redux、Redux-saga 和 React-Router 等工具集成在一起，使开发者可以更高效地管理应用的状态和异步操作

  1. 简化的 Redux 使用：

    • Dva 封装了 Redux 的复杂性，使得状态管理变得更加简单。它通过模型（Model）的概念来组织代码，将状态、异步请求和订阅整合在一起。

  2. 内置的副作用管理：

    • Dva 使用 Redux-saga 进行副作用管理，这使得处理异步操作（如数据获取）更加直观和可管理。

  3. 模型（Model）驱动：

    • Dva 的核心是模型，一个模型通常包括 state（状态）、reducers（同步操作）、effects（异步操作）和 subscriptions（订阅数据源）。

  4. 路由集成：

    • Dva 内置了 React Router 的支持，可以轻松地进行路由管理。

  5. 插件机制：

    • Dva 提供了插件机制，可以通过插件扩展 Dva 的功能，如状态持久化、错误处理等。

  6. 简化的 API：

    • Dva 提供了简化的 API，使得开发者可以更快速地上手和使用。

AHooks 是什么

官方文档 https://ahooks.js.org/zh-CN
AHooks 是由 Ant Design 团队开发的一个高质量且可靠的 React Hooks 库。它提供了一系列开箱即用的 Hooks，旨在提高开发效率，帮助开发者快速构建 React 应用。这些 Hooks 涉及状态管理、生命周期、DOM 操作、网络请求等常见的场景。以下是 AHooks 的一些特点和常用功能：

  1. 丰富的 Hooks 集合：

    • AHooks 提供了大量常用的 Hooks，例如 useRequest、useToggle、useLocalStorageState、useDebounceEffect 等，涵盖了数据请求、状态管理、性能优化等多个方面。

  2. **useRequest**：

    • 一个强大的 Hooks，用于处理异步请求。它支持请求的生命周期管理、并发控制、轮询、缓存等功能。

  3. 状态管理和 DOM 操作：

    • 提供了 useToggle、useBoolean、useSetState 等 Hooks，用于简化状态管理。

    • AHooks 的设计理念是易于使用和集成。即使是复杂的异步操作，也可以通过简单的 API 快速实现。
